JavaScript

About Comments

Sometimes when coding, programmers do not want everything they write to be used by the computer when running their program. This is when coders use comments! When programming, it is good practice to use comments to create English summaries of what the code means, so people who do not know how to code still know what the code does. Now we are going to code a comment!

About Functions

Functions are used to group blocks of code. They allow these blocks of code to be called, or carried out, multiple times within a computer program. Functions can be named using any name you want, but typically these names should reflect what the function does. In JavaScript, these names are written in camelCase. This means that the first letter of every word (starting on the second word of the function's name), and the rest of the letters are lowercase. Now we are going to create a function!

About Variables

Often, programmers need to keep track of numbers or letters. When this happens, they assign these values that they need to keep track of to their best friends, VARIABLES! Now we are going to create a variable!

About While Loops

When expressing actions that only occur during specific instances, coders use while loops. This means while one thing is true, we want something else to happen as well. Now we are going to code a while loop!

About Conditionals

Sometimes, coders want code to run only under certain circumstances. To do this, they use conditionals. These conditionals create certain conditions that must be true for the code inside of the conditionals to be carried out. Now we are going to code a conditional!

About For Loops

Programmers use for loops when they want to do the same thing a certain number of times. To keep track of the number of times the loop runs, we use a variable and then change this variable each time the loop runs. Now we are going to code a for loop!

Comments (Continued)

This is what a comment looks like:

//bye

Note how there are two forward slashes (without any spaces between them) at the beginning of the line. Everything on the rest of the line following those slashes is a comment. In this case, our comment is "bye." Get ready to code!

Functions (Continued)

This is what a basic function declaration looks like:

function myFunc() {}

The first part of a function declaration is the keyword "function." All programming languages have sets of keywords. These keywords cannot be used as variable, function, or class names. The name of this function is "myFunc()." Remember, the name of your function does not effect how your function will work. The parentheses are there to accept optional things called parameters. For now, don't worry about what those are, just know that they can go inside of the parentheses. The curly brackets ("{}") are where you put the code that belongs to the function. Get ready to code!

Variables (Continued)

This is what a basic variable declaration looks like:

var x = 5;

In JavaScript, just like algebra, we can use variables to hold values and we can use variables in equations. In a JavaScript variable declaration, you start with the keyword "var," to tell the computer you are creating a variable. Then, you choose the name of your variable. We chose "x." Remember, the name of the variable does not change how your variable will work. Then, we set "x" equal to "5". You can set a variable to integers, floating point numbers (numbers with decimals), and even strings (words). At the end of the line, we put a semicolon. Semicolons belong at the end of every line of code in JavaScript. Get ready to code!

While Loops (Continued)

This is what a basic while loop looks like:

while (1<2) {}

First, we start off with the keyword "while," which indicates that we are making a while loop. Then, we have parentheses. Inside of these parentheses we have the statement that must be true in order for the code inside the curly brackets ("{}") to run. Here, 1 must be less than 2 for the code to run. But 1 is always less than 2! Typically, variables are used (for example substitute 1 with x) in while loops so the statement isn't always true. To complete the while loop, put the code that you want to run inside of the curly brackets ("{}"). Get ready to code!

Conditionals (Continued)

This is what a basic conditional looks like:

if(0>1) {}

A conditional always starts off with the keyword "if." Then, we have parentheses. Inside of these parentheses we have the statement that must be true in order for the code inside the curly brackets ("{}") to run. Here, the condition is that 0 must be less than 1. 0 is always less than 1, so the code will always run. Typically, variables are used (for example, you could substitute 1 with x) when creating the conditions. To finish the conditional, put the code that you want to run inside the curly brackets ("{}"). Get ready to code!

For Loops (Continued)

This is what a basic for loop looks like:

for (var i = 0; i < 10; i++) {}

For loops always start off with the keyword "for." Then, we have parentheses. Inside of the parentheses, we create the variable "i" and set it equal to 0 ("var i = 0;."" Then we have the condition that i must be less than 10 ("i < 10;"). The last thing inside of the parentheses is "i++". This means that each time the code inside the for loop runs, 1 will be added to i. Due to this, our for loop will run 10 times! Note that in between each part of our parentheses we have semicolons. To fini sh the for loop, put the code that you want repeated inside the curly brackets ("{}"). Get ready to code!
